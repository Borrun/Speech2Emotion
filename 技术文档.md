# Speech2Emotion：基于因果时序建模与文本先验融合的语音情绪切换检测系统

## 摘要
本文面向实时情绪感知场景，提出并工程化实现了一个双链路语音情绪系统 Speech2Emotion。系统以因果时序卷积网络（Temporal Convolutional Network, TCN）为核心，联合建模帧级情绪类别、情绪强度等级与情绪切换边界概率；在此基础上，进一步引入文本先验驱动的在线融合检测器，实现“文本一次输入 + 音频分块流式输入”的低延迟边界检测。与仅依赖音频后处理的方案相比，文本先验融合可在保持在线性的同时提升边界触发的稳定性与可解释性。本文同时给出当前项目可复现实验链路、关键参数与可视化对齐机制。

**关键词**：情绪识别；情绪切换检测；因果 TCN；多任务学习；在线融合；语音-文本多模态

---

## 1. 引言
语音情绪识别（Speech Emotion Recognition, SER）在陪伴对话、客服质检与人机交互中具有重要价值。现有方法常聚焦于句级情绪分类，难以刻画连续语流中的情绪变化时刻。为解决该问题，本文将任务细化为帧级建模与切换点检测，并进一步探索文本语义对边界决策的辅助作用。

本项目的核心目标为：
1. 在 30fps 时间尺度输出稳定的帧级情绪状态；
2. 检测并解码情绪切换边界 `switch_frames`；
3. 在在线场景下融合文本先验与音频证据；
4. 通过统一看板实现音频、文本、区间与情绪曲线联动分析。

## 2. 方法

### 2.1 任务定义
给定语音序列 \(x\)，在帧率 \(f=30\,fps\) 下获得帧序列 \(x_t\)。模型输出：
- 情绪类别 \(y_t^{type}\)
- 情绪强度等级 \(y_t^{lvl}\in\{0,1,2,3,4,5\}\)
- 切换边界概率 \(p_t^{bnd}\in[0,1]\)

最终通过边界解码器从 \(p_t^{bnd}\) 得到切换点集合 \(S=\{s_i\}\)。

### 2.2 声学编码与多任务主干
系统采用因果 Log-Mel 特征（16kHz，30fps，50ms 窗长，`center=False`），并输入因果 TCN 主干。网络头部为三分支：
- `type_head`：情绪类别分类
- `lvl_head`：情绪等级分类
- `bnd_head`：边界二分类

训练目标为加权联合损失：
\[
\mathcal{L}=\lambda_{type}\mathcal{L}_{CE}^{type}+\lambda_{lvl}\mathcal{L}_{CE}^{lvl}+\lambda_{bnd}\mathcal{L}_{BCE}^{bnd}
\]

其中边界项采用正负样本重加权，以缓解边界稀疏带来的类别不平衡。

### 2.3 边界后处理解码
对帧级边界概率采用迟滞与确认窗口机制，参数包括：
- `thr_on`：进入候选边界阈值
- `thr_off`：退出候选边界阈值
- `confirm_win`：确认窗口长度
- `min_gap`：相邻边界最小间隔

该机制兼顾灵敏度与抗抖动能力，输出 `switch_frames` 及 `switch_times`。

### 2.4 文本先验在线融合
在融合链路中，文本一次性输入用于构建边界先验概率 \(p_t^{text}\)，音频流式输入提供 \(p_t^{audio}\)。融合概率定义为：
\[
p_t = w_{audio} p_t^{audio}+w_{text} p_t^{text}
\]
并在在线状态机中触发事件。当前 tuned 配置为：
- `chunk_frames=12`（约 0.4s/chunk，30fps）
- `w_audio=0.8`
- `w_text=0.2`
- `thr_on=0.62`
- `thr_off=0.42`
- `confirm_win=3`
- `min_gap=5`

## 3. 系统实现

### 3.1 双链路实现
- **链路 A（纯音频 TCN）**：
  `audio -> EmotionTCN -> boundary decode -> intervals`
- **链路 B（音频+文本融合 tuned）**：
  `text prior (one-shot) + audio boundary chunk streaming -> online fusion detector`

### 3.2 关键代码模块
- 模型与特征：
  - `models/model_emotion_tcn.py`
  - `models/featurizer.py`
- 推理与解码：
  - `infer/batch_infer.py`
  - `infer/postprocess.py`
  - `infer/interval_decode.py`
- 在线融合：
  - `online_emotion/text_prior.py`
  - `online_emotion/detector.py`
  - `tools/stream_fuse_predictions_with_text.py`
- 可视化：
  - `tools/make_alignment_dashboard.py`
  - `outputs/alignment_view/dashboard.html`

### 3.3 当前看板数据源对应
在 `outputs/alignment_view/dashboard.html` 中：
- `纯音频链（TCN）` -> `../test_alignment_view/dashboard_data.json`
- `音频+文本融合链（tuned）` -> `../test_alignment_view_stream_text_tuned_audio_textsrc/dashboard_data.json`

其中第二项为本项目最新实验：使用纯音频链文本替换融合链原文本后重跑融合得到。

## 4. 实验设置与复现

### 4.1 输入形式
- 音频：16kHz，特征按帧提取；融合阶段按 `chunk_frames=12` 流式输入
- 文本：转写一次性输入（非流式增量）

### 4.2 数据与输出
- 声学预测目录：`outputs/test_emotion_codes`
- 融合预测目录：`outputs/test_emotion_codes_stream_text_tuned_audio_textsrc`
- 区间目录：`outputs/test_interval_predictions_stream_text_tuned_audio_textsrc`
- 看板数据：`outputs/test_alignment_view_stream_text_tuned_audio_textsrc/dashboard_data.json`

### 4.3 复现命令
```bash
# 1) 纯音频链推理
/opt/anaconda3/envs/python3.8/bin/python infer/batch_infer.py \
  --wav_dir ./test \
  --ckpt ./outputs/ckpt_bnd_v1/best.pt \
  --out_dir ./outputs/test_emotion_codes \
  --device cpu

# 2) 用纯音频链文本重跑 tuned 融合
/opt/anaconda3/envs/python3.8/bin/python tools/stream_fuse_predictions_with_text.py \
  --pred_dir ./outputs/test_emotion_codes \
  --text_csv ./outputs/test_transcriptions_from_audio_chain.csv \
  --out_dir ./outputs/test_emotion_codes_stream_text_tuned_audio_textsrc \
  --chunk_frames 12 --w_audio 0.8 --w_text 0.2 \
  --thr_on 0.62 --thr_off 0.42 --confirm_win 3 --min_gap 5

# 3) 导出区间
/opt/anaconda3/envs/python3.8/bin/python tools/export_interval_predictions.py \
  --pred_dir ./outputs/test_emotion_codes_stream_text_tuned_audio_textsrc \
  --text_csv ./outputs/test_transcriptions_from_audio_chain.csv \
  --out_dir ./outputs/test_interval_predictions_stream_text_tuned_audio_textsrc
```

## 5. 结果与分析

### 5.1 现象总结
在当前实验设置下，融合链能够在多数样本上保持与纯音频链相近的切换数量，并在部分样本中减少误触发。该结果表明：文本先验在不改变声学主模型权重的前提下，能够作为有效的边界约束项。

### 5.2 工程优势
1. **可解释性强**：边界触发由显式阈值与窗口机制控制；
2. **部署成本低**：融合链可复用既有音频预测，不要求重训主模型；
3. **在线友好**：文本 one-shot + 音频 chunk 流式输入，满足低延迟需求。

### 5.3 局限性
1. 文本质量与分句策略会影响先验可靠性；
2. 不同数据域下 `thr_*` 与权重参数需要再标定；
3. 工具链在部分环境下存在 `torchaudio/OpenMP` 依赖差异。

---

## 6. 讨论
与端到端重型多模态模型相比，本文方案采用“主干声学建模 + 轻量先验融合”的分层结构，更强调可控性与工程可维护性。该策略特别适用于需要快速迭代、在线部署与可视化解释的一线应用场景。

---

## 7. 结论
本文给出了 Speech2Emotion 的完整实现与当前可复现实验配置。系统通过因果 TCN 多任务学习实现帧级情绪建模，并通过文本先验在线融合增强边界检测稳定性。最新看板版本已集成“纯音频文本替换后再融合”的实验结果，验证了在不重训主模型条件下进行输入侧变体优化的可行性。

未来工作将聚焦于跨域鲁棒性评估、参数自动寻优及更细粒度的多模态时序对齐机制。
