## 1. 因果梅尔频谱特征提取（CausalLogMelFeaturizer）

### 1.1 参数

| 参数 | 值 | 说明 |
|------|-----|------|
| `sample_rate` | 16000 Hz | 标准语音采样率 |
| `win_sec` | 0.05 s | 分析窗长 → `n_fft = round(16000 × 0.05) = 800` |
| `hop_sec` | 1/30 s | 帧移 → `hop_length = round(16000/30) ≈ 533` 采样点 |
| `n_mels` | 80 | Mel 滤波器数量 |
| `f_min` | 40 Hz | 最低频率 |
| `f_max` | 7600 Hz | 最高频率（上限 `min(7600, sr/2 - 100) = 7900`） |
| `center` | **False** | **关键**：不填充，防止未来信息泄露 |
| `power` | 2.0 | 幅度谱平方（功率谱） |

### 1.2 STFT（短时傅里叶变换）

输入时间段 `[t·H, t·H + N_fft)` 的功率谱：

$$S_t[k] = \left| \sum_{n=0}^{N_{fft}-1} w[n]\cdot x[n + t \cdot H] \cdot e^{-j2\pi kn/N_{fft}} \right|^2$$

其中：
- $H = 533$（hop_length）
- $N_{fft} = 800$（n_fft）
- $w[n]$：Hann 窗（torchaudio 默认）
- `center=False`：帧 $t$ 仅使用 $[t \cdot H, t \cdot H + N_{fft})$ 的过去/当前样本，**不使用未来样本**

### 1.3 Mel 滤波器组

三角形滤波器将 $K = N_{fft}/2+1 = 401$ 个频率仓映射到 $M = 80$ 个 Mel 频道：

$$\text{mel}(f) = 2595 \cdot \log_{10}\!\left(1 + \frac{f}{700}\right)$$

第 $m$ 个 Mel 滤波器：

$$H_m[k] = \begin{cases}
\dfrac{f[k] - f_{m-1}}{f_m - f_{m-1}} & f_{m-1} \le f[k] \le f_m \\[4pt]
\dfrac{f_{m+1} - f[k]}{f_{m+1} - f_m} & f_m \le f[k] \le f_{m+1} \\[4pt]
0 & \text{otherwise}
\end{cases}$$

Mel 滤波器输出：$M_t[m] = \sum_k H_m[k] \cdot S_t[k]$

### 1.4 对数压缩

$$\text{LogMel}_t[m] = \log\!\left(\max(M_t[m],\; 10^{-10})\right)$$

### 1.5 输出形状

$$\text{输入：} x \in \mathbb{R}^{1 \times N} \xrightarrow{\text{MelSpec}} \mathbb{R}^{1 \times 80 \times T} \xrightarrow{\text{log}} \xrightarrow{\text{squeeze+transpose}} \mathbb{R}^{T \times 80}$$

其中 $T = \lfloor (N - N_{fft}) / H \rfloor + 1$（`center=False` 时）

## 2. EmotionTCN 模型（因果时序卷积网络）

### 2.1 网络结构

```
Input: Mel [B, T, 80]
  │
  └─► transpose → [B, 80, T]
         │
         ▼
  ┌──────────────────────┐
  │  in_proj             │  Conv1d(80→128, k=1)  [B,128,T]
  └──────────┬───────────┘
             │
             ├── ResidualCausalBlock(d=1)   ← dilation=2^0=1
             ├── ResidualCausalBlock(d=2)   ← dilation=2^1=2
             ├── ResidualCausalBlock(d=4)   ← dilation=2^2=4
             ├── ResidualCausalBlock(d=8)   ← dilation=2^3=8
             ├── ResidualCausalBlock(d=16)  ← dilation=2^4=16
             └── ResidualCausalBlock(d=32)  ← dilation=2^5=32
                          │
                          ▼  [B,128,T]
             ┌────────────┼────────────┐
             ▼            ▼            ▼
   type_head(1×1)   lvl_head(1×1)  bnd_head(1×1)
   [B,T,7]          [B,T,6]        [B,T]  (sigmoid后)
```

### 2.2 CausalConv1d（因果卷积）

标准 Conv1d 若使用 `padding='same'` 会读取未来帧，因此改为左填充：

**填充量**：$P = (k-1) \cdot d$

**操作**：
```
x_padded = pad(x, (P, 0))        # 仅左填充，右侧不填充
y = Conv1d(x_padded, k, dilation=d)
```

输出帧 $t$ 的感受野：仅依赖 $[t - (k-1)d, \ldots, t]$，**不依赖未来帧**。

**单个 CausalConv1d 的感受野（帧数）**：

$$\text{RF} = 1 + (k-1) \cdot d$$

当 $k=3$：$\text{RF} = 1 + 2d$

### 2.3 ResidualCausalBlock

```
输入 x  [B,C,T]
  │
  ├── CausalConv1d(C→C, k=3, dilation=d)
  ├── ReLU
  ├── Dropout(0.1)
  ├── Conv1d(C→C, k=1)      ← 逐点映射
  └── ReLU
  │
  └── + x   ← 残差连接
  │
输出 x + net(x)  [B,C,T]
```

### 2.4 整体感受野

6个块，膨胀率 $d_i = 2^i$（$i=0,\ldots,5$），每块 $k=3$：

$$\text{RF}_{total} = 1 + \sum_{i=0}^{5} 2 \cdot (3-1) \cdot 2^i = 1 + 4 \sum_{i=0}^{5} 2^i = 1 + 4 \times 63 = 253 \text{ 帧}$$

对应时长：$253 \times (1/30) \approx 8.4$ 秒

### 2.5 输出头

| 输出头 | 操作 | 激活 | 形状 |
|--------|------|------|------|
| type_head | Conv1d(128→7, 1) + transpose | softmax（训练时 cross-entropy，推理时 argmax） | [B,T,7] |
| lvl_head | Conv1d(128→6, 1) + transpose | softmax（同上） | [B,T,6] |
| bnd_head | Conv1d(128→1, 1) + squeeze | **sigmoid** → $[0,1]$ | [B,T] |

### 2.6 情绪类别

| type_id | 0 | 1 | 2 | 3 | 4 | 5 | 6 |
|---------|---|---|---|---|---|---|---|
| 类别 | happy | sad | angry | fear | calm | confused | (reserved) |

| level_id | 0 | 1 | 2 | 3 | 4 | 5 |
|----------|---|---|---|---|---|---|
| 强度 | 最弱 | ← | ← | → | → | 最强 |

## 3. 文本边界先验（TextPrior）

### 3.1 候选边界提取（TextPriorBuilder）

从文本提取3类候选边界位置（归一化到 $[0,1]$）：

**3.1.1 标点边界**

```
文本长度 n = len(text)
对每个标点符号 text[i] ∈ {，,。！？!?；;：:}：
    p = (i+1) / n
    若 0.02 < p < 0.98：加入候选
```

**3.1.2 情感翻转边界**

```
将文本按标点切分为子句 clauses[]
每个子句打分：score = Σ(正向词命中) - Σ(负向词命中)
相邻子句若 score[i] × score[i+1] < 0（符号相反）：
    p = Σlen(clause[0..i]) / total_len
    若 0.02 < p < 0.98：加入候选
```

**3.1.3 Token 时序边界（可选）**

若提供 ASR token 时间戳：
```
对包含标点的 token：
    p = token.end_sec / max_end_sec
    若 0.02 < p < 0.98：加入候选
```

**去重**：对排序后的候选集，若相邻两点距离 $< 0.015$ 则保留前者：

$$|p_i - p_{i-1}| > \epsilon_{dedup} = 0.015$$

### 3.2 帧域映射

文本位置 $p \in [0,1]$ 到帧索引的映射：

$$\hat{t}(p) = \text{offset} + \text{scale} \times p$$

初始参数：
- $\text{offset} = 0$
- $\text{scale} = \text{init\_total\_frames}$（默认 180，或由 `total_sec × fps` 得到）

### 3.3 高斯先验概率

第 $t$ 帧的边界先验概率（各候选点高斯叠加）：

$$p_{text}(t) = \text{clip}\!\left(\sum_{i=1}^{N_c} \exp\!\left(-\frac{(t - \hat{t}(p_i))^2}{2\sigma^2}\right),\; 0,\; 1\right)$$

其中 $\sigma = 4.0$ 帧（约 133 ms），$N_c$ 为候选点数量。

### 3.4 自适应参数更新

每当音频边界探测器确认一个边界事件（帧 $t^*$），找最近未匹配候选 $p_j$：

$$\mu_j = \text{offset} + \text{scale} \times p_j$$

$$\text{err} = t^* - \mu_j$$

$$\text{offset} \mathrel{+}= \alpha_{offset} \cdot \text{err}$$

$$\text{scale} \mathrel{+}= \alpha_{scale} \cdot \frac{\text{err}}{p_j}, \quad \text{scale} \ge 20$$

默认：$\alpha_{offset} = 0.15$，$\alpha_{scale} = 0.05$

此机制使先验概率随对话进展动态校准到实际语速。

## 4. 在线边界融合检测器（OnlineBoundaryDetector）

### 4.1 线性融合

$$p_{fused}(t) = \text{clip}\!\left(w_{audio} \cdot p_{audio}(t) + w_{text} \cdot p_{text}(t),\; 0,\; 1\right)$$

默认：$w_{audio} = 0.8$，$w_{text} = 0.2$，$w_{audio} + w_{text} = 1.0$

### 4.2 边界状态机

```
状态变量：
  candidate_peak_idx   ← 当前候选的峰值帧（None 表示无候选）
  candidate_peak_fused ← 候选窗口内最大融合概率
  last_confirmed       ← 上一次确认的边界帧

对每帧 t，p = p_fused(t)：

  if candidate_peak_idx is None:          ─── 空闲状态
    if p >= thr_on:
      candidate_peak_idx   = t
      candidate_peak_fused = p
      → 进入候选状态

  else:                                    ─── 候选状态
    if p > candidate_peak_fused:
      candidate_peak_idx   = t            ← 峰值更新
      candidate_peak_fused = p

    if t >= candidate_peak_idx + confirm_win:   ← 确认条件1（超时）
      → 尝试发送边界事件（emit）

    elif p < thr_off:                           ← 确认条件2（提前回落）
      → 尝试发送边界事件（emit）

emit 逻辑：
  if t* - last_confirmed >= min_gap:       ← 间隔约束
    发出 BoundaryEvent(frame_idx=t*)
    last_confirmed = t*
    调用 text_prior.register_boundary(t*)  ← 触发自适应更新
    清空候选状态
  else:
    丢弃候选（min_gap 约束未满足）
```

### 4.3 参数说明

| 参数 | 默认值 | 作用 |
|------|--------|------|
| `thr_on` | 0.62 | 候选激活阈值 |
| `thr_off` | 0.42 | 提前确认（迟滞下沿） |
| `confirm_win` | 3 帧 | 超时确认等待帧数 |
| `min_gap` | 5 帧 | 边界最小间距（抑制密集误触发） |
| `w_audio` | 0.8 | 音频权重 |
| `w_text` | 0.2 | 文本权重 |

### 4.4 输出 BoundaryEvent

```python
BoundaryEvent:
  frame_idx : int    # 峰值帧索引
  t_sec     : float  # frame_idx / fps
  p_audio   : float  # 峰值时的音频概率
  p_text    : float  # 峰值时的文本先验概率
  p_fused   : float  # 峰值时的融合概率
```

## 5. 情绪平滑与段级统计（StreamingDecoder / C++ 端）

### 5.1 多数投票窗（Majority Vote）

维护长度为 `smooth_win` 的环形缓冲区，对窗口内的 `type_id` 和 `level_id` 分别取众数：

$$\hat{e}_t = \arg\max_{e} \sum_{\tau=\max(0,t-W+1)}^{t} \mathbb{1}[e_\tau = e]$$

### 5.2 迟滞确认（Hysteresis）

防止情绪在两个候选之间抖动：

```
candidate_label = majorityVote()
if candidate_label == current_label:
  cand_count = 0
elif candidate_label 连续出现 >= hysteresis 次:
  current_label = candidate_label
  cand_count = 0
else:
  cand_count++
```

### 5.3 区间累积投票（C++ SpeechEmotionEngine）

在边界事件之间累积所有帧的 type/level 投票：

```
interval_type_cnt_[7]   ← 各 type_id 累积计数
interval_level_cnt_[6]  ← 各 level_id 累积计数
interval_frame_cnt_     ← 当前区间帧数

on boundary:
  committed_type_  = argmax(interval_type_cnt_)
  committed_level_ = argmax(interval_level_cnt_)
  清零计数器

当前最优估计：
  if interval_frame_cnt_ >= kMinIntervalFrames(=5):
    cur = argmax(interval_type_cnt_)     ← 区间内渐进修正
  elif committed_type_ >= 0:
    cur = committed_type_               ← 使用上一边界提交值
  else:
    return                              ← 暖机期，无输出
```

### 5.4 发射延迟队列（Emit Delay Queue）

PCM 数据比音频播放提前到达，需要对齐：

```
环形缓冲区大小：kEmitDelay + 1 = 4
写指针：emit_write_
填充计数：emit_count_

每次 pushPCM：
  1. 若 emit_count_ == kEmitDelay+1：
       out = emit_buf_[emit_write_]   ← 读取最老的槽（将被覆盖）
       若 out 不同于上次输出：result.updated = true
  2. emit_buf_[emit_write_] = cur     ← 写入当前最优
  3. emit_write_ = (emit_write_+1) % (kEmitDelay+1)
  4. emit_count_ = min(emit_count_+1, kEmitDelay+1)
```

延迟量：$kEmitDelay = 3$ 个 chunk，每 chunk $\approx 8192/16000 \approx 0.5$ s，共约 **1.5 s**。

## 6. 文本情绪约束（TextEmotionConstraint）

### 6.1 情感极性计算（bilingual_sentiment_score）

**语言检测**：
```
CJK字符占比 > 15% → 中文
否则 → 英文（langdetect辅助确认）
```

**中文情感得分**：
$$s = \frac{n_{pos} - n_{neg}}{n_{pos} + n_{neg}} \in [-1, 1]$$

其中 $n_{pos}$，$n_{neg}$ 分别为正向、负向关键词命中数。若无关键词命中，使用 SnowNLP：

$$s = \text{SnowNLP.sentiments} \times 2 - 1$$

**英文情感得分**（NRCLex affect_frequencies）：

$$s = \frac{(\text{positive} + \text{joy} + \text{trust}) - (\text{negative} + \text{sadness} + \text{fear} + \text{anger})}{(\text{positive} + \text{joy} + \text{trust}) + (\text{negative} + \text{sadness} + \text{fear} + \text{anger})}$$

### 6.2 Layer 2：规则硬覆盖

| 条件 | 操作 |
|------|------|
| $s > +0.3$ 且 `type_id ∈ {SAD, ANGRY, FEAR}` | `type_id = HAPPY` |
| $s < -0.3$ 且 `type_id ∈ {HAPPY, CALM}` | `type_id = SAD` |
| 含问号 且 `level_id ≤ 2` 且 `type_id ≠ CONFUSED` | `type_id = CONFUSED` |

### 6.3 Layer 3：软融合

**文本情绪分布**（text_emotion_distribution）：

英文（NRCLex）：
$$d^{text}[i] = \sum_{c \in \text{NRC}} \text{af}[c] \cdot w_{c \to i}$$

其中 NRC→type 映射权重：

| NRC 类别 | type_id | 权重 |
|---------|---------|------|
| joy | HAPPY | 1.0 |
| positive | HAPPY | 0.4 |
| trust | CALM | 1.0 |
| anticipation | CALM | 0.6 |
| anger | ANGRY | 1.0 |
| disgust | ANGRY | 0.7 |
| sadness | SAD | 1.0 |
| negative | SAD | 0.4 |
| fear | FEAR | 1.0 |
| surprise | CONFUSED | 1.0 |

结构加权（语言无关）：
- 含问号 `?/？`：$d^{text}[\text{CONFUSED}] \mathrel{+}= 1.5$
- 含感叹号 `!/！`：$d^{text}[\text{HAPPY}] \mathrel{+}= 0.5$，$d^{text}[\text{ANGRY}] \mathrel{+}= 0.3$

归一化：$d^{text}[i] \leftarrow d^{text}[i] / \sum_j d^{text}[j]$

**混合决策**：

若 $d^{text}[\arg\max d^{text}] \ge \theta_{conf} = 0.5$ 且文本最优类别 $\ne$ 当前 `type_id`：

$$\tilde{d}[i] = w_{blend} \cdot d^{text}[i] + (1 - w_{blend}) \cdot \mathbb{1}[i = \text{type\_id}]$$

$$\text{type\_id}_{final} = \arg\max_i \tilde{d}[i]$$

默认 $w_{blend} = 0.35$，即文本占 35%，音频占 65%。

## 7. 完整数据流（逐步标注）

```
PCM chunk (float[], ~8192 samples @ 16kHz)
  │
  ▼ [CausalLogMelFeaturizer.push()]
Mel frames [N_new, 80]   (N_new = 新生成的帧，约10-15帧/chunk)
  │
  ▼ [EmotionTCN.forward()]
per-frame predictions:
  type_logits  [N_new, 7]  →  type_id  = argmax
  level_logits [N_new, 6]  →  level_id = argmax
  bnd_logits   [N_new]     →  p_audio  = sigmoid
  │
  ├─ p_audio[] ─────────────────────────────────────────────────────┐
  │                                                                  │
  ▼ [区间累积]                                                       ▼ [融合]
interval_type_cnt_[7] += type_id 命中                  p_text[]  ← TextPrior.probs_for_range()
interval_level_cnt_[6] += level_id 命中                p_fused[] = w_audio·p_audio + w_text·p_text
interval_frame_cnt_ += N_new                            │
  │                                                      ▼ [状态机]
  │                                          BoundaryEvent (若触发)
  │                                              │
  │◄─────────────── boundary=True ───────────────┘
  │
  ▼ [on boundary: 提交区间投票]
committed_type_  = argmax(interval_type_cnt_)
committed_level_ = argmax(interval_level_cnt_)
reset counters
  │
  ▼ [当前最优估计]
cur_type, cur_level  (区间内 rolling majority 或 committed)
  │
  ▼ [发射延迟队列]
emit_buf_[write_ptr] = {cur_type, cur_level}
output = emit_buf_[oldest_ptr]   (延迟 kEmitDelay=3 个 chunk)
  │
  ▼ [TextEmotionConstraint（Python端）]
Layer2: 硬规则覆盖
Layer3: 文本分布软融合
  │
  ▼
最终输出: type_id ∈ {0..5}, level_id ∈ {0..5}
```

## 8. C++ 端与 Python 端对比

| 模块 | Python（离线/评估） | C++（在线/生产） |
|------|---------------------|-----------------|
| 特征提取 | `CausalLogMelFeaturizer` (torchaudio) | `MelFeaturizer` (自实现) |
| 模型推理 | PyTorch `.forward()` | ONNX Runtime |
| 边界解码 | `OnlineBoundaryDetector` + `TextPrior` | `StreamingDecoder`（仅音频） |
| 情绪平滑 | 滑动窗多数投票 + 迟滞 | 区间累积 + 提交 + 发射延迟 |
| 文本融合 | `TextEmotionConstraint` Layer2/3 | 暂未集成（由 Planner 控制） |
| 输出格式 | JSON (type_id, level_id, events[]) | `SpeechEmotionEngine::Result` |
