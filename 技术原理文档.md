# Speech2Emotion 流式方案技术原理文档

## 1. 目标与输入输出

### 1.1 目标
构建可在线运行的情绪切换检测与情绪状态跟踪系统，支持：
- 文本一次输入（text one-shot）
- 音频分块流式输入（10~15帧/chunk）
- 每个 chunk 输出：
  - 当前帧情绪 `type/level`（raw 与 stable）
  - 是否触发边界事件
  - 当前段 ID 与段主情绪

### 1.2 输入
- 音频主模型预测结果（每帧）：`type_id`, `level_id`, `boundary_p`
- 文本：句子级转写（一次提供）

### 1.3 输出
- `boundary_events`：融合边界事件帧
- `frames[]`：逐帧 `raw / stable_causal / stable`
- `chunks[]`：逐 chunk 摘要，包含边界、当前帧状态、段状态、chunk 文本与文本特征

---

## 2. 整体链路

### 2.1 音频主链（离线预测）
`audio -> Causal Log-Mel -> EmotionTCN -> per-frame(type, level, boundary_p)`

### 2.2 在线融合链（本方案）
1. 文本构建边界先验 `p_text(t)`
2. 音频流式提供 `p_audio(t)`
3. 融合：
\[
p_{fused}(t)=w_{audio}p_{audio}(t)+w_{text}p_{text}(t)
\]
4. 迟滞 + 确认窗解码边界事件（`thr_on/thr_off/confirm_win/min_gap`）
5. 情绪状态平滑与迟滞（因果）得到 `stable_causal`
6. 固定延迟前后文修正（lookahead）得到最终 `stable`
7. 用边界事件切段，段内多数投票得到 `segment major emotion`

---

## 3. 关键算法

### 3.1 边界检测（在线状态机）
- 进入候选：`p_fused >= thr_on`
- 候选峰值更新：取候选窗口内最大 `p_fused`
- 确认条件：
  - 达到 `confirm_win`，或
  - `p_fused < thr_off`（提前确认）
- 事件间隔约束：`min_gap`

该机制抑制抖动和密集误触发。

### 3.2 情绪平滑（因果）
对 `raw(type_id, level_id)` 执行短窗多数投票 + 迟滞确认：
- `smooth_win`：窗口长度
- `emo_hysteresis`：切换确认帧数

输出 `stable_causal`，保证在线实时可用。

### 3.3 前后文修正（固定延迟）
为避免“单 chunk 决策偏差”，对每帧再做固定延迟修正：
- 当前帧使用 `[t-back, t+lookahead]` 窗口多数投票
- 结果写入 `stable`

说明：
- `lookahead=0` 为纯因果
- `lookahead>0` 引入有限未来信息，提升稳定性但增加等量延迟

### 3.4 文本绑定 chunk（新增）
将文本 token 对齐到帧轴后，为每个 chunk 生成：
- `chunk_text`
- `text_features`：
  - `n_tokens`
  - `punct_count`
  - `pos_hits`
  - `neg_hits`
  - `sentiment_score`

这些特征当前进入输出与分析链，可进一步用于动态调权（如自适应 `w_text`）。

## 4. 可视化原理（outputs.html）

页面：`outputs/stream_online/outputs.html`

### 4.1 波形画布（已合并时间轴）
同一画布叠加：
- 音频波形
- 按情绪类型着色的 `stable level` 曲线
- 边界事件竖线
- 当前 chunk 高亮
- 播放指针

### 4.2 文本条
- 文本 token 按帧对齐铺在时间轴
- 随播放时间高亮当前 token

### 4.3 情绪眼部线条画布
- 从 `emotions/*.txt` 读取模板曲线
- 按当前 `stable type/level` 映射模板帧
- 绘制眼部线条（重点两条加粗）

### 4.4 Chunk 详情区
展示每个 chunk 的核心输出：
- `boundary_events`
- `current_frame.raw/stable`
- `segment.id/major`
- `chunk_text/text_features`

---

## 5. 关键参数（当前默认）

### 5.1 边界融合
- `w_audio=0.8`
- `w_text=0.2`
- `thr_on=0.62`
- `thr_off=0.42`
- `confirm_win=3`
- `min_gap=5`

### 5.2 流式组织
- `chunk_min=10`
- `chunk_max=15`

### 5.3 情绪稳定
- `smooth_win=5`
- `emo_hysteresis=3`
- `future_lookahead=8`

- 在线主脚本：`tools/stream_online_emotion.py`

## 7. 运行方式

### 7.1 生成流式结果（单文件）
```bash
python tools/stream_online_emotion.py \
  --pred_json outputs/test_emotion_codes/web_20260309.122708.683.json \
  --text_csv outputs/test_transcriptions_from_audio_chain.csv \
  --chunk_min 10 --chunk_max 15 \
  --smooth_win 5 --emo_hysteresis 3 --future_lookahead 8 \
  --w_audio 0.8 --w_text 0.2 \
  --thr_on 0.62 --thr_off 0.42 --confirm_win 3 --min_gap 5 \
  --out outputs/stream_online/web_20260309.122708.683.stream_online.json
```

### 7.2 查看可视化
```bash
python -m http.server 8000
```
打开：
`http://localhost:8000/outputs/stream_online/outputs.html`
